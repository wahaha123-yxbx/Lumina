## 全局约定

### 响应格式

```
// 成功
{ "code": 0, "data": T, "message": "ok" }

// 失败
{ "code": 1, "data": null, "message": "错误描述" }
```

### 时间格式

所有时间字段均为 **毫秒级时间戳**（`Date.now()`），非 ISO 字符串。

### ID 前缀

| 前缀 | 表 | 示例 |
|------|-----|------|
| `u_` | users | `u_kf8a3b2c` |
| `p_` | posts | `p_kf8a3b2d` |
| `c_` | comments | `c_kf8a3b2e` |
| `proj_` | projects | `proj_kf8a3b2f` |
| `task_` | tasks | `task_kf8a3b2g` |
| `app_` | applications | `app_kf8a3b2h` |
| `n_` | notifications | `n_kf8a3b2i` |
| `s_` | starDustRecords | `s_kf8a3b2j` |
| `conv_` | conversations | `conv_kf8a3b2k` |
| `msg_` | messages | `msg_kf8a3b2l` |

### 认证方式

请求头: `Authorization: Bearer <token>`

认证中间件不阻断请求，仅自动注入 `req.userId`。部分接口需自行校验登录态。

## 1. 认证 API

### 1.1 注册

```
POST /api/auth/register
```

**请求体：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `studentId` | string | ✅ | 学号，唯一 |
| `password` | string | ✅ | 密码 |
| `nickname` | string | | 昵称，默认 `"星尘旅人"` |

**响应：**

```json
{
  "code": 0,
  "data": {
    "token": "lm_kf8a...",
    "user": { /* User 对象，不含密码字段 */ }
  },
  "message": "注册成功"
}
```

**错误：**
- `400` 学号和密码不能为空
- `400` 该学号已注册

---

### 1.2 登录

```
POST /api/auth/login
```

**请求体：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `studentId` | string | ✅ | |
| `password` | string | ✅ | |

**响应：** 同注册，`message: "登录成功"`

**错误：**
- `400` 学号未注册
- `400` 密码错误

---

### 1.3 获取当前用户

```
GET /api/auth/me
```

**请求头：** `Authorization: Bearer <token>`

**响应：** `{ code: 0, data: User }`

**错误：**
- `401` 未登录
- `404` 用户不存在

## 2. 帖子 API

### 2.1 帖子列表

```
GET /api/posts?tag=生活
```

| 参数 | 位置 | 必填 | 说明 |
|------|------|------|------|
| `tag` | query | | 分类筛选，不传或传 `"全部"` 返回所有 |

**排序：** `isTop` 置顶优先 → `createTime` 倒序

**响应：** `{ code: 0, data: Post[] }`

---

### 2.2 热榜帖子

```
GET /api/posts/hot
```

**排序公式：** `likeCount×2 + commentCount×3 + starDustCount×5 + viewCount×0.1`

---

### 2.3 帖子详情

```
GET /api/posts/:id
```

**行为：** `viewCount` 自动 +1

**错误：** `404` 帖子不存在

---

### 2.4 创建帖子

```
POST /api/posts
```

**请求体：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `title` | string | ✅ | |
| `content` | string | | 默认 `""` |
| `authorId` | string | ✅ | |
| `tag` | string | | 默认 `"生活"` |

**行为：** 发布者 `postCount + 1`，`starDustBalance + 2`

---

### 2.5 点赞/取消点赞

```
POST /api/posts/:id/like
```

**请求体：** `{ "userId": "u_xxx" }`

**行为：** 已点赞则取消，未点赞则点赞（触发通知 `type: "like"`）

## 4. 打赏 API

### 4.1 打赏帖子

```
POST /api/posts/:id/reward
```

**请求体：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `fromUserId` | string | ✅ | 打赏者 |
| `amount` | number | ✅ | 星尘数量 |
| `message` | string | | 默认 `"支持优质内容"` |

**行为：** 余额扣减 → 接收者 `totalStarDustReceived` 增加 → 等级联动 → `starDustCount` 增加 → 写入 `starDustRecords`

**错误：**
- `404` 用户或帖子不存在
- `400` 星尘余额不足

### 等级对照表

| 条件 | level | levelTitle |
|------|-------|------------|
| totalReceived ≥ 10000 | 6 | 星海领主 |
| totalReceived ≥ 5000 | 5 | 星尘大师 |
| totalReceived ≥ 2000 | 4 | 星光使者 |
| totalReceived ≥ 500 | 3 | 星辰旅者 |
| totalReceived ≥ 100 | 2 | 星尘学徒 |
| 其他 | 1 | 星尘新芽 |

## 11. 任务 API

### 11.1 任务列表

```
GET /api/projects/:id/tasks
```

**排序：** `createTime` 倒序

**响应：** `{ code: 0, data: Task[] }`

---

### 11.2 创建任务

```
POST /api/projects/:id/tasks
```

**请求体：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `title` | string | ✅ | |
| `description` | string | | 默认 `""` |
| `assigneeId` | string | | 不传即为未分配 |
| `priority` | string | | 默认 `"medium"` |
| `deadline` | number\|null | | 毫秒时间戳 |

**行为：**
1. 创建任务（status 默认为 `"todo"`）
2. `project.taskCount += 1`
3. `project.updateTime` 刷新

**priority 枚举：** `low` / `medium` / `high` / `urgent`

---

### 11.3 更新任务

```
PUT /api/tasks/:id
```

**请求体：** 所有字段可选

| 字段 | 类型 | 说明 |
|------|------|------|
| `title` | string | |
| `description` | string | |
| `status` | string | `todo` / `inProgress` / `done` |
| `assigneeId` | string | 传 `null` 或 `""` 清空负责人 |
| `priority` | string | |
| `deadline` | number\|null | |

**状态变更联动：**
- 任务从非 `done` → `done`：`project.completedTaskCount += 1`
- 任务从 `done` → 非 `done`：`project.completedTaskCount -= 1`
- `project.updateTime` 刷新

**负责人变更：** 自动查 `users` 表更新 `assigneeName` 和 `assigneeAvatar`

---

### 11.4 删除任务

```
DELETE /api/tasks/:id
```

**行为：**
- `project.taskCount -= 1`
- 如果任务是 `done` 状态，`project.completedTaskCount -= 1`

## 12. 队员招募 API

### 12.1 申请加入项目

```
POST /api/projects/:id/apply
```

**请求体：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `userId` | string | ✅ | |
| `message` | string | | 申请理由，默认 `"我想加入这个项目！"` |

**校验：**
1. `userId` 不能为空
2. 不能重复申请同一项目（pending 状态下）
3. 已是成员不能再申请

**行为：**
1. 创建 application 记录
2. 通知项目队长（`type: "apply"`）

---

### 12.2 查看项目申请列表

```
GET /api/projects/:id/applications
```

**用途：** 队长查看申请列表

**排序：** `createTime` 倒序

---

### 12.3 审批申请

```
PUT /api/applications/:id
```

**请求体：**

```json
{ "status": "approved" }
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `status` | string | ✅ | `"approved"` 或 `"rejected"` |

**`approved` 行为：**
1. 将申请人加入 `project.members`（role: `"成员"`）
2. `user.projectCount += 1`
3. 人数满员 → `project.status = "active"`
4. 通知申请人（`type: "approve"`）

**`rejected` 行为：** 仅更新 `application.status`，不做其他联动

---

### 12.4 我的申请记录

```
GET /api/users/:id/applications
```

查询该用户发起过的所有申请。

---

### 12.5 我的项目

```
GET /api/users/:id/projects
```

查询该用户参与的所有项目（通过 `members` 数组匹配）。

**排序：** `updateTime` 倒序

## 13. AI 匹配 API

### 13.1 为项目匹配队员

```
GET /api/projects/:id/match
```

**匹配逻辑（关键词法）：**
1. 提取项目关键词：`tags + category + title 分词 + description 前10个词`
2. 排除已在项目中的用户
3. 对每个用户，计算其 `nickname + bio + major + skills` 与项目关键词的重合度
4. 得分 > 0 的按分数排序，取 Top 10

**响应：**

```json
{
  "code": 0,
  "data": [
    {
      "userId": "u_005",
      "nickname": "算法小王",
      "avatar": "🦊",
      "bio": "热爱深度学习",
      "major": "计算机科学",
      "level": 3,
      "levelTitle": "星辰旅者",
      "projectCount": 2,
      "matchScore": 5
    }
  ]
}
```

---

### 13.2 为用户推荐项目

```
GET /api/users/:id/recommend-projects
```

**逻辑反向：** 用用户的 `nickname + bio + major + skills` 去匹配 `recruiting` 状态的项目。

**响应：** `Project[]`，每个项目带上 `matchScore` 字段
